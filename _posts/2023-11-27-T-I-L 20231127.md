### REST API 디자인 가이드 핵심 사항
- 1. URI는 정보의 자원을 표현해야 한다. ex. 회원, 게시글
- 2. 자원에 대한 행위는 HTTP 메서드로 표현한다.
- (x) URI에 행위를 넣기
- (x) 마지막 문자로 슬래시(/) 포함
- (x) URI에 밑줄 시용(밑줄 대신 하이픈 - 사용 해야함, 이것도 불가피한 경우만)
- (x) URI에 대문자 사용
- (x) URI에 파일 확장자 포함 ex. /member/{memberId}/profile.jpg
- HTTP method의 종류
